# Task Note

## Goal

Maintain a GitHub Pages personal blog focused on security-query learning notes, source-reading notes, failed-construction retrospectives, and future research questions.

## Boundary

Do not disclose unpublished core protocol ideas, patent-sensitive implementation details, private raw data, credentials, or local personal paths in posts. Blog posts should be written as learning notes and research retrospectives.

## Editing workflow

- Drafts may go in `drafts/`.
- Published posts go in `_posts/YYYY-MM-DD-title.md`.
- Run simple Markdown/link checks before publishing when possible.
